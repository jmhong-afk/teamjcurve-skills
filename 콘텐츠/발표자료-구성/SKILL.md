# 발표자료 구성

## 역할

발표자가 "무엇을 어떤 순서로 말할 것인가"를 결정하는 단계에서 시간을 절약해준다. 주제와 청중만 알면 슬라이드 흐름, 각 슬라이드의 핵심 메시지, 발표자 노트까지 한 번에 생성하기 때문에 빈 화면 앞에서 막히는 시간을 없앤다. 생성된 구성안을 베이스로 발표자가 내용을 검토·수정하면 실제 PPT 제작 시간이 크게 줄어든다.

## 입력

사용자에게 받아야 하는 것:
- **발표 주제**: 발표의 핵심 메시지 또는 제목 (예: "AI 도입 사례 공유", "2025 J-Curve 성과 보고")
- **대상 청중**: 누구에게 발표하는지 (예: 팀 내부, 고객사 임원, 투자자, 교육 수강생)
- **발표 시간**: 전체 발표 길이 (예: 10분, 30분, 1시간)
- **전달 목표**: 발표 후 청중에게 남겨야 할 것 (예: 설득, 정보 전달, 행동 유도, 보고)
- **보유 내용** (선택): 이미 가진 데이터, 사례, 키워드 등. 없으면 생략해도 무방

## 처리 순서

1. **청중과 목표 파악** — 같은 주제라도 청중과 목표에 따라 강조점이 완전히 달라지기 때문에 가장 먼저 확인한다. 임원 보고용은 결론 먼저, 교육용은 맥락 먼저.

2. **전체 흐름(스토리라인) 설계** — 슬라이드 수와 섹션을 정한다. 발표 시간 기준으로 슬라이드 수를 산정한다 (1분당 1~1.5장 기준). 오프닝 → 문제 제기 → 본론 → 결론·행동 요청 구조를 기본으로 사용하는 이유는 청중이 "왜 이 발표를 들어야 하는지"를 초반에 납득해야 이후 내용이 흡수되기 때문이다.

3. **슬라이드별 내용 작성** — 각 슬라이드에 제목, 핵심 메시지 한 줄, 포함할 내용 bullet, 발표자 노트를 작성한다. 핵심 메시지를 한 줄로 강제하는 이유는 슬라이드당 하나의 생각만 전달해야 청중의 이해가 흐트러지지 않기 때문이다.

4. **시각화 힌트 추가** — 텍스트 대신 도표·그래프·이미지가 효과적인 슬라이드를 표시한다. 직접 디자인은 하지 않지만 발표자가 PPT 제작 시 방향을 잡을 수 있도록 돕는다.

## 출력 형식

```
# [발표 제목] 구성안

## 기본 정보
- 대상: [청중]
- 시간: [총 발표 시간] / 슬라이드 [총 장수]장
- 목표: [발표 후 청중이 해야 할 것 / 느껴야 할 것]

---

## 슬라이드 구성

### 슬라이드 1 — [슬라이드 제목]
**핵심 메시지:** [이 슬라이드에서 청중에게 남겨야 할 한 문장]
**내용:**
- [bullet 1]
- [bullet 2]
**발표자 노트:** [말로 풀어야 할 보충 설명 또는 전환 멘트]
**시각화 힌트:** [이미지/도표/텍스트 중 추천 형식]

### 슬라이드 2 — ...
(반복)

---

## 전체 흐름 요약
[오프닝 → 섹션1 → 섹션2 → ... → 마무리 한 줄 요약]
```

## 예시

**예시 1:**
입력: 주제=AI 업무 자동화 사례 발표, 청중=비개발자 팀원 10명, 시간=15분, 목표=AI 도구 실제 사용 시작 유도
출력:
```

## 기본 정보
- 대상: 비개발자 팀원 10명
- 시간: 15분 / 슬라이드 12장
- 목표: 발표 후 팀원 중 최소 3명이 AI 도구를 실제로 써보게 만들기

---

### 슬라이드 1 — 여러분의 하루에서 가장 반복되는 일은?
**핵심 메시지:** 우리는 매일 자동화 가능한 일에 시간을 쓰고 있다
**내용:**
- 청중이 공감할 반복 업무 예시 나열 (이메일 초안, 회의록, 보고서 요약)
- "이 중 몇 개나 AI가 대신할 수 있을까요?" 질문으로 오프닝
**발표자 노트:** 손을 들게 하거나 간단한 체크인으로 시작하면 집중도 높아짐
**시각화 힌트:** 반복 업무 아이콘 그리드 이미지
```

**예시 2:**
입력: 주제=3개월 프로젝트 성과 보고, 청중=대표 및 임원 4명, 시간=10분, 목표=다음 분기 예산 승인, 보유 내용=매출 120% 달성·고객 만족도 4.7·이탈률 감소 데이터
출력:
```
### 슬라이드 1 — 한 줄 요약: 목표 초과 달성
**핵심 메시지:** 설정 목표 대비 120% 달성, 다음 분기 확장 투자 적기
**내용:**
- 매출 목표 vs 실적 숫자 (대비표)
- 핵심 지표 3개 (매출·만족도·이탈률) 스냅샷
**발표자 노트:** 첫 장에 결론부터 — 임원은 결과를 먼저 보고 싶어 한다
**시각화 힌트:** 목표 vs 실적 막대 그래프
```

## 한계 및 주의사항

- **사실 검증 불가**: AI가 생성한 내용(통계, 사례)은 실제 데이터가 아니다. 발표에 쓸 수치와 사례는 반드시 발표자가 실제 데이터로 교체해야 한다.
- **디자인 제작 불가**: 이 스킬은 텍스트 구성안만 생성한다. 실제 PPT/Keynote/Figma 파일 제작은 발표자가 한다.
- **청중 반응 예측 한계**: 특정 조직의 문화, 발표 당일 분위기, 의사결정자의 선호는 AI가 알 수 없다. 발표 전 관련자 피드백을 받을 것.
- **발표자 목소리 부재**: 생성된 발표자 노트는 예시일 뿐이다. 발표자 고유의 말투와 스타일로 반드시 재작성할 것.
