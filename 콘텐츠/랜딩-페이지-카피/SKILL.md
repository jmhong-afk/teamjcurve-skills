# 랜딩 페이지 카피

## 역할

교육 프로그램의 랜딩 페이지는 방문자가 "나한테 맞는 프로그램인가"를 판단하고 신청으로 이어지게 만드는 세일즈 문서다. 정보 전달이 목적인 모집 공고와 달리, 랜딩 페이지는 공감 → 신뢰 → 행동 유도 순서로 흐르는 퍼널 구조가 핵심이다. 이 스킬은 그 구조에 맞게 섹션별 카피를 완성형으로 생성한다.

## 입력

사용자에게 받아야 하는 것:
- **프로그램 기본 정보**: 이름, 대상, 기간, 형식(온라인/오프라인/혼합), 인원, 비용
- **핵심 변화(outcomes)**: 이 프로그램을 마쳤을 때 참가자에게 생기는 구체적인 변화나 결과물 (예: "AI 툴로 업무 자동화 파이프라인 구축", "포트폴리오 3개 완성")
- **대상 페르소나**: 누구를 위한 프로그램인지, 어떤 상황에 있는 사람인지 (예: "기획자인데 데이터 분석이 안 되는 사람", "창업 초기 혼자 마케팅 해야 하는 대표")
- **차별점**: 비슷한 프로그램 대비 이 프로그램만의 특징 (예: 소규모 코호트, 실무 프로젝트 중심, 강사 1:1 피드백)
- **사회적 증거(선택)**: 이전 기수 후기, 수료율, 수료생 커리어 변화 등

## 처리 순서

1. **페르소나 공감 헤드라인 작성** — 방문자가 첫 3초 안에 "이거 나 얘기네"를 느껴야 스크롤을 내린다. 프로그램 이름보다 참가자의 현재 상황과 불편함을 먼저 건드린다.

2. **핵심 가치 제안(Value Proposition) 정리** — 공감 다음에 "그래서 이 프로그램은 무엇을 해결해주는가"를 한 문단으로 명확히 한다. 기능 나열이 아니라 변화 중심으로 작성한다.

3. **대상자 명시 섹션** — "이런 분께 맞습니다 / 이런 분께는 맞지 않습니다"를 구체적으로 나열한다. 대상을 좁힐수록 맞는 사람의 신뢰가 높아진다.

4. **커리큘럼/프로그램 구조 요약** — 세부 커리큘럼보다 "이 과정에서 무엇을 만들거나 경험하는가"를 중심으로 작성한다. 읽는 사람이 수료 후 모습을 상상할 수 있어야 한다.

5. **사회적 증거 삽입** — 수강 후기, 수료율, 수료생 성과를 구체적 숫자나 인용문으로 배치한다. 증거가 없으면 운영 철학이나 강사 소개로 대체한다.

6. **FAQ 섹션** — 신청을 망설이게 하는 가장 흔한 3~5가지 이유를 먼저 찾아 Q&A 형태로 해소한다.

7. **CTA(행동 유도) 문구** — 버튼 텍스트 + 마감 긴박감 + 신청 후 다음 단계를 함께 작성한다.

## 출력 형식

섹션 순서대로 완성형 카피를 생성한다. 각 섹션은 실제 페이지에 붙여 넣을 수 있는 수준으로 작성한다.

```
[헤드라인]
(페르소나 공감형 주 카피, 1~2줄)
(서브 카피, 2~3줄로 프로그램이 주는 변화 한 줄 요약)

[이 프로그램은]
(핵심 가치 제안 단락, 3~5문장)

[이런 분께 맞습니다]
- ...
- ...
- ...

[이런 분께는 맞지 않습니다]
- ...
- ...

[프로그램 구조]
(기간 / 형식 / 핵심 경험 2~4개를 짧게)

[수강생 이야기]
(후기 인용 또는 성과 수치, 없으면 운영 철학 단락)

[자주 묻는 질문]
Q: ...
A: ...
(3~5개)

[지금 신청하기]
버튼 문구: ...
마감/정원 안내: ...
신청 후 안내: ...
```

## 예시

**예시 1:**

입력:
- 프로그램명: AI 실무 부트캠프 5기
- 대상: 기획/마케터 중 AI 툴을 써보고 싶은데 어디서 시작할지 모르는 직장인
- 기간: 4주, 주 2회 온라인
- 핵심 outcomes: 업무 자동화 파이프라인 1개 구축, GPT/Claude 실무 활용법 체득
- 차별점: 매 세션마다 실제 내 업무에 바로 적용하는 실습 위주, 강사 Slack 피드백

출력:
```
[헤드라인]
AI 써보고 싶은데, 뭐부터 해야 할지 모르겠다면

ChatGPT 계정은 있는데 결국 검색창처럼만 쓰고 있다면, 이 프로그램이 그 막막함의 출발점을 바꿔드립니다.

[이 프로그램은]
AI 실무 부트캠프는 이론보다 실습이 먼저입니다. 매 세션에서 배운 툴을 그날 내 업무에 바로 대입해보는 방식으로, 4주 후에는 반복 업무를 줄여주는 자동화 파이프라인 하나를 직접 만들게 됩니다. 강사에게 Slack으로 피드백을 받으며 진행하기 때문에, 막히는 지점을 혼자 해결하지 않아도 됩니다.

[이런 분께 맞습니다]
- 기획·마케팅 업무를 하면서 AI 툴이 내 일에 어떻게 연결되는지 감이 안 잡히는 분
- 유튜브 강의는 봤는데 실제 적용이 안 되는 분
- 바쁜 직장인 일정에 맞는 주 2회 온라인 수업을 원하는 분

[이런 분께는 맞지 않습니다]
- 이미 LLM API를 실무에 연동해본 개발자
- 이론 중심 강의를 선호하는 분

[프로그램 구조]
4주 / 주 2회 온라인 라이브 / 매 세션 실습 과제 / Slack 피드백 채널
핵심 경험: GPT·Claude 프롬프트 설계 → 반복 업무 분석 → 자동화 파이프라인 설계 → 최종 발표

[자주 묻는 질문]
Q: 비개발자도 따라갈 수 있나요?
A: 네. 코딩 없이 진행하는 세션이 대부분입니다. 개발 지식이 없어도 자동화 파이프라인을 구성할 수 있도록 설계했습니다.

Q: 녹화본 제공 되나요?
A: 모든 세션은 녹화 후 1주일 이내 제공됩니다.

[지금 신청하기]
버튼 문구: 5기 신청하기
마감: 선착순 20명 / 모집 마감 4월 15일
신청 후: 신청 완료 후 24시간 내 오리엔테이션 안내 이메일 발송
```

**예시 2:**

입력:
- 프로그램명: 창업자 마케팅 집중반 3기
- 대상: 초기 창업자, 혼자 마케팅 해야 하는 1인 대표
- 기간: 2일 오프라인 워크샵
- outcomes: SNS 콘텐츠 캘린더 1개월치 완성, 타깃 페르소나 정의 완료
- 차별점: 당일 실습으로 결과물 가져가기, 참가자 수 12명 한정

출력:
```
[헤드라인]
마케팅 해야 하는 건 아는데, 뭘 올려야 할지 매번 막힌다면

2일 만에 한 달치 콘텐츠 계획을 완성하고 돌아갑니다.

[이 프로그램은]
창업자 마케팅 집중반은 강의를 듣는 자리가 아닙니다. 내 브랜드의 타깃 페르소나를 직접 정의하고, 그 페르소나에 맞는 1개월치 콘텐츠 캘린더를 워크샵 안에서 완성합니다. 12명 한정으로 운영하기 때문에 즉각적인 피드백과 참가자 간 크로스 리뷰가 가능합니다.

[이런 분께 맞습니다]
- 창업 초기 혼자 SNS 마케팅을 담당하고 있는 대표
- 콘텐츠 방향을 잡지 못해 올리다 멈추다를 반복하는 분
- 이론보다 당일 결과물을 원하는 분

[이런 분께는 맞지 않습니다]
- 마케팅 전담 팀이 있어 직접 실행하지 않는 분
- 온라인 수업을 선호하는 분

[프로그램 구조]
2일 오프라인 / 서울 / 참가자 12명 한정
Day 1: 페르소나 정의 + 채널 전략 수립
Day 2: 콘텐츠 캘린더 제작 + 1:1 피드백

[자주 묻는 질문]
Q: 사업 초기라 브랜드가 정리가 안 된 상태인데 참여 가능한가요?
A: 오히려 이 단계가 적기입니다. 방향이 없는 상태에서 방향을 잡는 것이 이 워크샵의 목적입니다.

Q: 결과물을 못 완성하면 어떻게 되나요?
A: 미완성 결과물은 워크샵 후 2주 이내 Slack 채널에서 추가 피드백을 받을 수 있습니다.

[지금 신청하기]
버튼 문구: 3기 신청하기 (12명 한정)
마감: 선착순 마감 / 잔여석 확인 후 신청
신청 후: 신청 확인 후 장소 안내 및 사전 설문 발송
```

## 한계 및 주의사항

- **실제 수강 후기 문구는 AI가 생성하지 않는다.** 가상의 후기는 신뢰를 오히려 낮춘다. 실제 후기가 없으면 운영 철학이나 강사 소개로 대체해야 한다.
- **전환율은 카피만으로 결정되지 않는다.** 페이지 디자인, 로딩 속도, 신청 폼 UX도 전환에 영향을 미친다. 카피 완성 후 디자이너와 UX 검토를 반드시 거쳐야 한다.
- **가격 민감도는 사람이 판단해야 한다.** 비용 정보를 어디에 얼마나 노출할지, 할인 문구를 넣을지는 운영 전략에 따라 달라지므로 AI 단독으로 결정하지 않는다.
- **SEO 최적화는 별도 작업이다.** 이 스킬은 카피 작성에 집중하며, 메타 태그·키워드 밀도 최적화는 포함하지 않는다.

## 조직 활용 가이드

Claude Code에서 쓰는 법:
1. `@workspace/skills/콘텐츠/랜딩-페이지-카피/SKILL.md` 참조
2. 아래 프롬프트로 바로 시작:
   ```
   랜딩 페이지 카피 작성해줘.
   프로그램명: [이름]
   대상: [페르소나]
   기간/형식: [기간, 온/오프라인]
   핵심 outcomes: [수료 후 변화]
   차별점: [다른 프로그램 대비 특징]
   후기/증거: [있으면 입력, 없으면 생략]
   ```

팀 공유 방법:
- `skills/` 폴더를 GitHub 레포에 올려두면 팀원 누구나 clone 후 동일하게 사용
- 신규 팀원 온보딩 시 스킬 레포 clone이 첫 번째 단계
