# 이메일 초안

## 역할

반복적으로 작성하는 업무 이메일(안내, 요청, 감사, 사과, 확인 등)을 빠르게 초안으로 만든다.
어조, 구조, 길이를 받는 사람과 목적에 맞게 조정하기 때문에 "어떻게 시작하지?"에서 멈추는 시간을 줄인다.
초안을 받아 빠르게 검토·수정하는 방식으로 쓰면 이메일 한 통당 작성 시간을 절반 이하로 단축할 수 있다.

## 입력

사용자에게 받아야 하는 것:
- **받는 사람**: 관계와 호칭 (예: 수강생 전체, 협력사 담당자 김OO 대리, 강사 이OO 님)
- **목적**: 이메일로 전달하려는 핵심 (예: 다음 주 일정 변경 안내, 강의 자료 제출 요청, 워크샵 참가 감사)
- **핵심 내용**: 날짜, 링크, 금액, 조건 등 빠지면 안 되는 정보
- **어조** (선택): 공식적 / 친근하게 / 간결하게 — 언급 없으면 받는 사람 관계에서 자동 판단

## 처리 순서

1. **받는 사람과 목적 파악** — 같은 "감사 메일"이라도 수강생에게 쓰는 것과 기업 파트너에게 쓰는 것은 어조가 달라야 하기 때문에 관계를 먼저 확인한다.
2. **어조 결정** — 명시된 경우 그대로 따르고, 없으면 관계(상하 / 동급 / 고객)와 목적(요청 / 안내 / 사과)을 보고 공식·친근·간결 중 선택한다.
3. **구조 설계** — 이메일은 인사 → 목적 한 문장 → 본문 → 행동 요청(CTA) → 마무리 순서가 읽는 사람 입장에서 가장 빠르게 파악되기 때문에 이 흐름을 기본으로 쓴다.
4. **초안 생성** — 제목 포함. 본문은 단락을 짧게 유지해 모바일에서도 읽기 편하게 한다.
5. **수정 포인트 안내** — 실제 날짜, 이름, 링크 등 사람이 직접 채워야 할 자리를 `[대괄호]`로 표시한다.

## 출력 형식

```
제목: [이메일 제목]

안녕하세요, [받는 사람 호칭]

[목적 한 문장 — 이 이메일을 왜 보내는지 바로 알 수 있어야 한다]

[본문 — 핵심 정보 전달. 2~4 단락 이내]

[행동 요청 — 언제까지 무엇을 해달라는지 명확하게]

감사합니다.
[보내는 사람 이름 / 소속]
```

## 예시

**예시 1:**
입력: 다음 주 화요일 워크샵 시작 시간이 오전 10시에서 오후 2시로 변경됨. 수강생 전체에게 안내. 장소는 그대로.
출력:
```
제목: [워크샵명] 시작 시간 변경 안내 (3/[일] 화요일)

안녕하세요, 수강생 여러분

다음 주 화요일 워크샵 시작 시간이 변경되어 안내드립니다.

- 기존: 오전 10:00
- 변경: 오후 2:00
- 장소: 변동 없음 ([장소명])

일정 조정에 참고해 주시기 바랍니다. 문의 사항이 있으시면 언제든지 연락 주세요.

감사합니다.
[이름 / 소속]
```

**예시 2:**
입력: 강사 이OO 님께 다음 기수 강의 자료를 이번 주 금요일까지 제출해 달라고 요청. 친근하게.
출력:
```
제목: [기수명] 강의 자료 제출 요청 (금요일까지)

안녕하세요, 이OO 님!

다음 기수 준비가 시작되어 연락드립니다.

이번 주 금요일([날짜])까지 강의 자료를 [제출 방법/링크]으로 공유해 주시면 감사하겠습니다.

바쁘신 중에 번거롭게 해드려 죄송하고, 궁금한 점 있으시면 편하게 말씀해 주세요!

감사합니다.
[이름]
```

## 한계 및 주의사항

- **실제 날짜·이름·링크는 AI가 채울 수 없다** — 맥락에 제공되지 않은 고유 정보는 `[대괄호]`로 비워 두므로 반드시 직접 확인 후 채울 것.
- **조직 내부 관행은 반영 불가** — 특정 팀의 이메일 서명 형식, 내부 호칭 규칙 등은 사람이 직접 맞춰야 한다.
- **발송 전 전체 검토 필수** — 초안은 출발점이다. 어조가 맞는지, 빠진 정보가 없는지 사람이 최종 확인 후 발송한다.
