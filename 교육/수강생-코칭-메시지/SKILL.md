# 수강생 코칭 메시지 작성

## 역할

코칭 메시지는 강사가 수강생에게 보내는 가장 직접적인 개입 수단이다. 잘 쓴 한 통의 메시지가 수강생의 동기와 방향을 바꾼다. 이 스킬은 수강생 정보를 입력받아 그 사람에게만 해당하는 구체적인 피드백과 다음 행동을 포함한 코칭 메시지를 생성한다.

## 입력

사용자에게 받아야 하는 것:
- **수강생 정보**: 이름, 직군/배경, 현재 기수 또는 수강 단계 (예: "김민수, 마케터 3년차, 6기 3주차")
- **현재 상태**: 최근 제출물, 참여도, 막힌 부분, 잘하고 있는 것 (예: "과제 제출 늦음, 질문은 많음, 실행보다 분석에 강함")
- **코칭 목적**: 격려인지, 방향 수정인지, 중간 점검인지, 다음 단계 안내인지 명시 (예: "중간 점검 + 실행 독려")
- **톤 (선택)**: 따뜻한 격려 / 직접적인 피드백 / 공식적 / 편안한 대화체 (기본값: 따뜻하되 구체적)

## 처리 순서

1. **수강생 상태 파악** — 입력 정보에서 강점, 약점, 현재 위치를 먼저 정리한다. 맥락 없이 메시지를 쓰면 공허한 격려가 된다.

2. **핵심 메시지 1개 결정** — 이 메시지에서 수강생이 가져가야 할 핵심이 무엇인지 한 문장으로 정한다. 여러 메시지를 동시에 전달하려 하면 아무것도 전달되지 않는다.

3. **구체적 근거 포함** — "잘하고 있어요"는 의미 없다. 어떤 행동/결과를 보고 그렇게 말하는지 명시한다. 수강생은 자신이 관찰되고 있다는 것을 느낄 때 신뢰를 갖는다.

4. **다음 행동 1가지 제안** — 추상적 조언 대신 이번 주에 할 수 있는 구체적인 한 가지 행동을 제안한다. 선택지가 많으면 아무것도 안 한다.

5. **열린 마무리** — 질문이나 반응을 유도하는 문장으로 마무리한다. 일방향 메시지보다 대화를 여는 편이 코칭 효과가 높다.

## 출력 형식

```
[수신자 이름]님,

[관찰된 사실 1~2문장 - 구체적 행동/결과 언급]

[핵심 피드백 또는 격려 - 이유 포함, 2~3문장]

[다음 행동 제안 - 구체적이고 실행 가능한 1가지]

[열린 마무리 - 질문 또는 지지 표현]

[발신자 이름/서명]
```

## 예시

**예시 1:**
입력:
- 수강생: 이서연, PM 2년차, 5기 4주차
- 현재 상태: 매주 과제 제출 정시, 발표 때 말이 빠르고 결론이 불분명함, 질문이 깊고 논리적
- 코칭 목적: 발표 방식 개선 + 강점 확인
- 톤: 따뜻하되 직접적

출력:
```
서연님,

4주 내내 과제를 정시에 내셨고, 매번 질문이 날카로워서 잘 보고 있습니다. 특히 지난주 경쟁사 분석에서 놓치기 쉬운 변수를 짚어낸 건 인상적이었어요.

한 가지 나눠드리고 싶은 건 발표할 때입니다. 서연님은 생각이 빠른 만큼 말도 빠른데, 듣는 사람 입장에서는 결론을 찾기 어려울 때가 있어요. 서연님이 가진 분석력을 상대가 제대로 받아들이게 하려면, 결론을 먼저 말하고 근거를 붙이는 순서로 연습해보시면 좋겠습니다.

이번 주 발표 준비할 때, 첫 문장을 "결론은 ○○입니다"로 시작하는 버전을 한 번 써보시겠어요?

해보고 어떤지 알려주시면 같이 다듬어 볼게요.

지민 드림
```

**예시 2:**
입력:
- 수강생: 박준혁, 개발자 5년차, 7기 2주차
- 현재 상태: 과제 1회 미제출, 슬랙 반응은 있음, 기술적 역량은 탁월하나 비즈니스 관점 연결을 어려워함
- 코칭 목적: 이탈 방지 + 방향 재설정
- 톤: 편안한 대화체

출력:
```
준혁님,

슬랙에서 다른 분들 과제에 리액션 남기시는 거 보면서, 관심은 있으신데 뭔가 걸리는 게 있으신 것 같다는 느낌이 들었어요.

지난주 과제 못 내신 거 체크했는데, 괜찮으세요? 기술적으로는 제일 앞서 계신 분이라, 비즈니스 관점 연결하는 부분이 낯설게 느껴지시는 건 아닌가 싶었어요. 그 부분은 사실 개발 배경 있으신 분들이 가장 빠르게 잡을 수 있는 영역인데, 처음엔 낯선 게 맞습니다.

이번 주 과제, 완성도 0순위로 두지 말고 일단 생각나는 대로 60% 짜리라도 내보시겠어요? 같이 보면서 맞춰나가면 되니까요.

부담 있으면 먼저 DM 주셔도 됩니다.

지민 드림
```

## 한계 및 주의사항

- **수강생의 실제 감정 상태는 파악 불가**: 텍스트 정보만으로 작성하므로, 보내기 전에 최근 대화나 표정·분위기를 직접 확인한 강사가 톤을 최종 조정해야 한다.
- **개인 민감 정보 포함 금지**: 건강, 가족 상황 등 민감한 맥락을 Claude에 입력하지 말 것. 그 정보는 메시지에 직접 반영하지 않는 편이 낫다.
- **발신 전 반드시 사람이 검토**: 생성된 메시지는 초안이다. 수강생과의 실제 관계 맥락을 아는 사람이 최종 확인 후 발송한다.
