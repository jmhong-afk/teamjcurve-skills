# 교보재 제작

## 역할

강의나 워크샵 준비 시 매번 처음부터 자료를 만드는 시간을 줄여준다. 학습 목표에서 역산하여 수강생이 가장 빠르게 이해할 수 있는 구조로 자료를 초안화한다. 교육 설계 원칙(목표 → 내용 → 활동 → 확인)을 자동으로 적용하기 때문에 누락 없이 일관된 퀄리티를 유지할 수 있다.

## 입력

사용자에게 받아야 하는 것:
- **강의 주제**: 무엇을 가르치는지 (예: "린 스프린트 방법론 소개", "OKR 작성법")
- **대상 수강생**: 누가 배우는지 + 사전 지식 수준 (예: "스타트업 팀장급, OKR 처음 접하는 분들")
- **학습 목표**: 이 자료를 보고 나면 수강생이 무엇을 할 수 있어야 하는지 1~3개 (예: "OKR의 O와 KR을 스스로 구분하고 작성할 수 있다")
- **자료 형식**: 핸드아웃 / 실습지 / 개념 정리 시트 / 체크리스트 중 선택 (복수 가능)
- **분량 기준**: 수업 시간 또는 A4 페이지 수 (예: "90분 강의용", "A4 2장 이내")

입력이 불완전하면 학습 목표만 먼저 확인하라. 나머지는 추론 가능하다.

## 처리 순서

1. **학습 목표 재정의** — 입력받은 목표를 "수강생이 ~을 할 수 있다" 형태의 행동 동사로 환원한다. 목표가 명확해야 이후 모든 내용 선택의 기준이 생긴다.

2. **핵심 개념 추출** — 목표 달성에 반드시 필요한 개념만 선별한다. 관련 있어 보이는 내용을 전부 넣으면 자료가 무거워지고 수강생이 핵심을 놓친다.

3. **자료 구조 설계** — 형식(핸드아웃/실습지 등)에 맞게 섹션을 배치한다. 개념 → 예시 → 실습 → 정리 순서가 인지 부하를 최소화하기 때문에 기본 구조로 사용한다.

4. **내용 채우기** — 각 섹션별로 수강생 수준에 맞는 언어로 작성한다. 전문 용어는 처음 등장할 때 한 줄로 정의한다.

5. **실습/확인 문제 생성** — 학습 목표와 1:1 대응하는 실습 문제 또는 확인 질문을 붙인다. 목표와 무관한 문제는 제거한다.

## 예시

**예시 1:**
입력: 주제=린 스프린트 회고, 대상=팀장급 10명(스크럼 경험 없음), 목표=KPT 회고를 스스로 진행할 수 있다, 형식=실습지, 분량=A4 1장

출력:
```
# 린 스프린트 회고 실습지
팀명: _________ | 참여자: _________________________

## 오늘의 목표
이 실습을 마치면 KPT 프레임워크로 팀 회고를 직접 진행할 수 있습니다.

## KPT란?
Keep(계속할 것) / Problem(문제였던 것) / Try(다음에 시도할 것)
회고의 핵심은 판단이 아닌 관찰입니다.

## Step 1: 개인 작성 (10분)
지난 스프린트를 떠올리며 각 항목에 3개씩 적어보세요.

Keep (잘 됐던 것):
1. _______________  2. _______________  3. _______________

Problem (어려웠던 것):
1. _______________  2. _______________  3. _______________

Try (다음엔 이렇게):
1. _______________  2. _______________  3. _______________

## Step 2: 팀 합산 (15분)
포스트잇에 옮겨 붙이고, 가장 많이 겹친 항목 TOP 3를 선택하세요.

팀 TOP 3 Problem: _______________________________________
팀 TOP 3 Try: ___________________________________________

## 다음 스프린트 액션 아이템
담당자 지정까지 완료해야 "회고 완료"입니다.

| 액션 | 담당자 | 완료 기한 |
|------|--------|-----------|
|      |        |           |
|      |        |           |
```

**예시 2:**
입력: 주제=OKR 작성법, 대상=스타트업 팀원(OKR 처음), 목표=Objective와 KR을 구분하고 팀 OKR 초안을 작성할 수 있다, 형식=핸드아웃, 분량=A4 2장

출력: Objective(방향) vs KR(측정 가능 결과) 구분 개념 정의 → 나쁜 예/좋은 예 대비 → 팀별 초안 작성 칸 포함 2페이지 핸드아웃 초안

## 한계 및 주의사항

- **조직 내부 맥락 반영 불가**: 팀 고유의 용어, 이전 기수 피드백, 실제 사례는 사람이 직접 추가해야 한다. AI는 일반적인 예시만 생성한다.
- **난이도 보정 불가**: 수강생의 실제 수준은 직접 가르쳐본 사람만 안다. 생성된 초안의 난이도가 적절한지 반드시 검토할 것.
- **디자인/레이아웃**: 이 스킬은 내용 초안만 생성한다. Figma, Canva, Google Slides 등으로의 옮김은 사람이 한다.
- **법적·저작권 검토**: 외부 사례나 인용이 포함될 경우 출처와 사용 범위를 사람이 확인해야 한다.
