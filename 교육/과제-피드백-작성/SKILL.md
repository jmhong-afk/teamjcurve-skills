# 과제-피드백-작성

## 역할

수강생 한 명 한 명의 과제 내용을 분석해 구체적이고 개인화된 피드백 문서를 생성한다.
"잘했어요" 같은 추상적 평가 대신, 과제의 어떤 부분이 왜 좋은지, 어떤 부분을 어떻게 개선하면 되는지를 명확하게 전달하기 때문에 수강생 입장에서 다음 행동이 바로 보인다.
강사 입장에서는 수강생 수가 많아도 품질 균일한 피드백을 빠르게 생성할 수 있다.

## 입력

사용자에게 받아야 하는 것:
- **과제 주제/기준**: 이번 과제가 무엇인지, 어떤 역량을 평가하려 했는지 (예: "자신의 고객 세그먼트를 정의하고 Why Now를 작성하는 과제")
- **수강생 이름 + 과제 내용**: 한 명 이상. 텍스트로 붙여넣기, 또는 "이름: 내용" 형식으로 제공
- **피드백 톤** (선택): 따뜻하게 / 직접적으로 / 코치처럼 — 명시 없으면 따뜻하되 직접적으로 작성
- **분량 기준** (선택): 수강생당 몇 줄 수준인지 — 명시 없으면 200~300자 내외로 생성

## 처리 순서

1. **과제 기준 파악** — 어떤 역량/결과물을 기대했는지 먼저 정의해야, 잘한 것과 부족한 것의 판단 기준이 생긴다.
2. **수강생 과제 분석** — 각 과제에서 잘 된 점(구체적 근거 포함)과 개선 포인트(이유 + 방향 포함)를 각각 추출한다. 추상적 판단이 아닌 과제 텍스트의 특정 부분을 인용하거나 참조해야 개인화 피드백이 된다.
3. **피드백 초안 작성** — 잘한 점 → 개선 포인트 → 다음 액션 순서로 작성한다. 이 순서가 수강생이 방어적이지 않게 피드백을 받아들이는 데 유리하기 때문이다.
4. **톤 조율** — 요청된 톤에 맞게 문장을 다듬는다. 기본은 "이 부분이 좋았어요 + 이 부분을 이렇게 바꾸면 더 좋아져요" 구조.
5. **일괄 정리** — 여러 수강생이면 이름별로 구분해 한 문서에 정리한다. 붙여넣기 바로 쓸 수 있도록 수강생 이름을 헤더로 구분한다.

## 출력 형식

수강생별로 아래 구조를 반복한다. 읽는 사람이 빠르게 내용을 파악할 수 있도록 잘한 점과 개선점을 분리한다.

```
---
[수강생 이름]

잘한 점:
[과제에서 구체적으로 잘 된 부분 + 왜 좋은지 이유]

개선할 점:
[어떤 부분이 부족한지 + 어떻게 바꾸면 더 나아지는지 방향 제시]

다음 액션:
[수강생이 바로 해볼 수 있는 구체적인 한 가지 행동]
---
```

## 예시

**예시 1:**
입력:
- 과제 주제: 자신의 타겟 고객을 한 문장으로 정의하기
- 수강생: 김민준 / "저는 20대 직장인 중 사이드프로젝트를 시작하고 싶지만 시간이 없는 사람들을 타겟으로 합니다."
- 톤: 따뜻하게

출력:
```
---
김민준

잘한 점:
타겟 고객의 상황(직장인, 사이드프로젝트 욕구)과 핵심 장벽(시간 부족)을 하나의 문장에 잘 담았습니다. 읽는 사람이 고객이 누구인지 바로 그려지는 수준입니다.

개선할 점:
"20대 직장인"이라는 인구통계 정보만으로는 세그먼트가 너무 넓습니다. 어떤 직종인지, 어느 정도 경력인지, 어떤 종류의 사이드프로젝트를 원하는지를 좁히면 실제 타겟에 더 가까워집니다.

다음 액션:
"시간이 없는 이유"가 무엇인지(야근, 육아, 긴 통근 등)를 하나 골라 문장에 추가해 보세요.
---
```

**예시 2:**
입력:
- 과제 주제: Why Now 작성 (지금 이 시장이 열리는 이유)
- 수강생: 이서연 / "AI가 발전하고 있어서 지금이 좋은 시점입니다."
- 톤: 직접적으로

출력:
```
---
이서연

잘한 점:
AI라는 트렌드를 Why Now 근거로 연결한 방향 자체는 맞습니다.

개선할 점:
"AI가 발전하고 있다"는 너무 일반적인 진술입니다. 심사자나 고객은 이 문장을 100개의 다른 스타트업에서도 읽습니다. 내 서비스에만 해당하는 구체적인 기술 변화나 데이터(예: GPT-4 이후 API 비용 90% 감소, 특정 법규 통과 등)를 근거로 연결해야 설득력이 생깁니다.

다음 액션:
"AI 발전" 대신 내 서비스가 지금 가능해진 구체적인 이유 하나를 숫자나 사건으로 표현해 보세요.
---
```

## 한계 및 주의사항

- **AI는 과제의 맥락 외부 정보를 모른다**: 수강생이 이전 세션에서 어떤 고민을 했는지, 개인적 사정이 있는지는 알 수 없다. 사람이 필요 시 보완해야 한다.
- **과제 내용이 너무 짧으면 개인화 한계**: 한두 줄짜리 과제는 인용할 내용이 없어 피드백이 일반적이 될 수 있다. 수강생에게 충분한 분량을 제출하게 유도하는 것이 선행되어야 한다.
- **최종 발송 전 사람이 반드시 검토**: AI가 생성한 피드백은 초안이다. 특히 수강생 이름이 내용과 엇갈리지 않았는지, 민감한 표현이 없는지 확인 후 발송할 것.
