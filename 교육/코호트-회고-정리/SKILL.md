# 코호트 회고 정리

## 역할

코호트가 끝날 때마다 반복되는 피드백 수집과 회고 정리를 자동화한다. 흩어진 수강생 의견을 패턴별로 묶고, 운영자가 다음 기수에 바로 적용할 수 있는 액션 아이템 형태로 변환해 주기 때문에, 원문 텍스트를 읽고 분류하는 수작업 시간을 대폭 줄인다.

## 입력

사용자에게 받아야 하는 것:
- **피드백 원문**: 수강생 설문 응답, 오픈 채팅 메시지, 수기 메모 등 형식 무관 (예: "세션 3가 너무 빨랐어요", "실습 시간이 부족합니다")
- **코호트 기본 정보**: 기수 번호, 총 수강생 수, 운영 기간, 주요 커리큘럼 (예: "7기, 20명, 4주, AI 실무 활용")
- **강사/운영자 메모** (선택): 운영 중 직접 느낀 문제점이나 특이사항

## 처리 순서

1. **피드백 분류** — 원문을 그대로 나열하면 패턴이 보이지 않기 때문에, 먼저 주제별(콘텐츠 / 운영 / 환경 / 강사)로 묶는다.

2. **감성 레이블링** — 각 피드백에 긍정/부정/중립을 붙인다. 이후 Keep/Problem 분류의 근거가 되며, 비율을 보면 전체 만족도를 수치 없이도 파악할 수 있다.

3. **KPT 도출** — 분류된 피드백에서 패턴을 읽어 Keep(유지), Problem(문제), Try(시도)로 정리한다. Try는 반드시 Problem에서 논리적으로 이어지도록 작성한다.

4. **액션 아이템 생성** — Try를 "누가 / 무엇을 / 언제까지" 형태로 구체화한다. 추상적인 개선 의지가 아닌 실행 가능한 단위로 쪼갠다.

5. **요약 카드 생성** — 한 페이지 분량의 기수 요약을 만든다. 다음 기수 기획 시 참고 자료로 쓰이고, 팀 공유도 이 카드 하나로 가능하도록 한다.

## 출력 형식

```
# [N기] 코호트 회고 — [운영 기간]

## 기수 개요
- 수강생: N명 | 기간: [날짜] | 커리큘럼: [주제]

## 피드백 요약
- 긍정 N건 / 부정 N건 / 중립 N건
- 주요 키워드: [키워드1], [키워드2], [키워드3]

## Keep (유지할 것)
- [긍정 패턴 1]: 근거가 된 피드백 인용
- [긍정 패턴 2]: 근거가 된 피드백 인용

## Problem (개선이 필요한 것)
- [문제 1]: 근거가 된 피드백 인용
- [문제 2]: 근거가 된 피드백 인용

## Try (다음 기수에 시도할 것)
- [시도 1] → Problem [문제 1] 해결
- [시도 2] → Problem [문제 2] 해결

## 액션 아이템
| 항목 | 담당 | 기한 |
|------|------|------|
| [구체적 실행 내용] | [담당자] | [N기 시작 전] |

## 다음 기수 참고 메모
[운영자가 특히 주의할 점 2~3줄]
```

## 예시

**예시 1:**

입력:
```
코호트: 7기, 수강생 18명, 4주 과정, AI 실무 도구 활용
피드백:
- "ChatGPT 실습 시간이 너무 짧았어요. 따라가기 바빴습니다."
- "강사님이 질문에 잘 답해주셔서 좋았어요."
- "3주차 과제가 갑자기 어려워졌어요."
- "슬랙 공지가 늦게 와서 헷갈렸습니다."
- "실습 예제가 실제 업무랑 연결돼서 이해하기 쉬웠어요."
```

출력:
```
# 7기 코호트 회고 — 4주 과정

## 기수 개요
- 수강생: 18명 | 기간: 4주 | 커리큘럼: AI 실무 도구 활용

## 피드백 요약
- 긍정 2건 / 부정 3건 / 중립 0건
- 주요 키워드: 실습 시간, 난이도 급등, 공지 지연

## Keep
- 강사 Q&A 응답 품질: "강사님이 질문에 잘 답해주셔서 좋았어요."
- 실무 연계 예제: "실습 예제가 실제 업무랑 연결돼서 이해하기 쉬웠어요."

## Problem
- 실습 시간 부족: "ChatGPT 실습 시간이 너무 짧았어요."
- 3주차 난이도 급등: "3주차 과제가 갑자기 어려워졌어요."
- 운영 공지 지연: "슬랙 공지가 늦게 와서 헷갈렸습니다."

## Try
- 실습 파트를 세션당 10분 추가 확보 → Problem: 실습 시간 부족
- 3주차 과제에 중간 난이도 단계 1개 삽입 → Problem: 난이도 급등
- 슬랙 공지 D-1 예약 발송 규칙화 → Problem: 공지 지연

## 액션 아이템
| 항목 | 담당 | 기한 |
|------|------|------|
| 세션별 실습 시간 재배분 (커리큘럼 수정) | 강사 | 8기 시작 2주 전 |
| 3주차 과제 난이도 단계 추가 | 강사 | 8기 시작 1주 전 |
| 슬랙 공지 D-1 예약 프로세스 문서화 | 운영 | 즉시 |
```

**예시 2:**

입력:
```
코호트: 3기, 수강생 12명, 2일 집중 워크샵, 전략 기획 입문
강사 메모: 오전 세션은 집중도가 높았으나 오후에 급격히 떨어짐. 그룹 발표 시간이 예상보다 30분 초과.
피드백: "그룹 토론이 제일 유익했어요", "오후에 너무 졸렸어요", "발표 시간이 너무 길어졌어요", "실제 케이스 스터디가 좋았어요"
```

출력:
```
# 3기 코호트 회고 — 2일 집중 워크샵

## 기수 개요
- 수강생: 12명 | 기간: 2일 | 커리큘럼: 전략 기획 입문

## 피드백 요약
- 긍정 2건 / 부정 2건 / 중립 0건
- 주요 키워드: 그룹 토론, 오후 집중도 저하, 발표 시간 초과

## Keep
- 그룹 토론 방식: "그룹 토론이 제일 유익했어요."
- 실제 케이스 스터디 활용: "실제 케이스 스터디가 좋았어요."

## Problem
- 오후 집중도 저하: 강사 메모 + "오후에 너무 졸렸어요."
- 발표 시간 30분 초과: 강사 메모 + "발표 시간이 너무 길어졌어요."

## Try
- 오후 첫 30분에 짧은 활동형 워밍업 삽입 → Problem: 오후 집중도 저하
- 팀당 발표 시간 타이머 도입 및 시간 상한 고지 → Problem: 발표 시간 초과

## 액션 아이템
| 항목 | 담당 | 기한 |
|------|------|------|
| 오후 오프닝 5분 활동 설계 (퀴즈 또는 브레인스토밍) | 강사 | 4기 시작 전 |
| 발표 타이머 운영 룰 문서화 | 운영 | 즉시 |
```

## 한계 및 주의사항

- **피드백 맥락 해석 불가**: 같은 "실습이 어려웠다"도 난이도 문제인지 시간 문제인지는 추가 질문 없이 판단하기 어렵다. Problem 분류가 애매한 항목은 "해석 필요" 표시를 붙이고 운영자가 최종 판단해야 한다.
- **수치 기반 만족도 없음**: NPS나 별점 데이터가 없으면 감성 비율은 정성적 추정이다. 숫자가 필요하면 설문 도구에서 직접 추출해야 한다.
- **액션 아이템 담당자 지정**: AI는 조직 구조를 모르기 때문에 담당자 이름은 운영자가 채워야 한다.
- **반복 패턴 추적 불가**: 이번 기수 데이터만 보기 때문에 "지난 3기수에서 반복된 문제"는 파악하지 못한다. 누적 비교가 필요하면 이전 회고 보고서를 함께 입력해야 한다.
