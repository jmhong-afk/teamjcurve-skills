# 강의 커리큘럼 설계

## 역할

과정명·대상·기간·핵심 목표를 입력받아 주차(또는 회차)별 세션 구성표를 생성한다. 각 세션에는 학습 목표, 주요 내용, 학습 활동, 준비물이 포함되어 있어 강사가 바로 강의 계획서 초안으로 활용할 수 있다. 커리큘럼 설계에서 가장 시간이 많이 드는 "전체 흐름을 어떻게 배분할까"의 문제를 AI가 초안으로 해결하고, 사람은 내용의 깊이와 실제 실습 설계에 집중할 수 있다.

## 입력

사용자에게 받아야 하는 것:
- **과정명**: 무엇을 가르치는 과정인지 (예: "AI 활용 실무 부트캠프", "마케팅 기초 강의")
- **총 기간 및 세션 수**: 주 단위 또는 회차 수 (예: "6주 12회차", "4주 8회차", "하루짜리 워크샵 5세션")
- **수강 대상**: 경험·배경·수준 (예: "마케팅 경력 3년 미만 실무자", "코딩 비전공 대학생")
- **핵심 목표**: 과정 종료 후 수강생이 할 수 있어야 하는 것 1~3가지 (예: "ChatGPT로 업무 자동화 1가지 구현")
- **선택 입력** — 다음이 있으면 커리큘럼 품질이 올라감:
  - 사전 지식 수준 (없음/기초/중급)
  - 강조하고 싶은 주제 또는 빼고 싶은 주제
  - 실습/강의 비중 선호도

## 처리 순서

1. **최종 목표 확정** — 역설계를 위해 과정이 끝났을 때 수강생이 무엇을 할 수 있어야 하는지를 먼저 구체화한다. 목표가 흐리면 커리큘럼도 흐려진다.

2. **구간 분할** — 총 세션 수를 "도입(Intro) → 핵심(Core) → 심화(Advanced) → 통합(Capstone)" 4구간으로 대략 2:4:3:1 비율로 나눈다. 비율은 기간·목표에 따라 유연하게 조정하되, 이유를 함께 제시한다.

3. **세션별 주제 배정** — 각 구간 안에서 세션 주제를 배정한다. 앞 세션이 뒤 세션의 전제 지식이 되는 의존 관계를 고려해 순서를 결정한다. 의존 관계가 없는 주제는 참여도가 낮아지는 중반부에 실습 비중을 높여 배치한다.

4. **세션 상세 작성** — 각 세션마다 학습 목표(동사로 시작)·주요 내용·학습 활동·준비물을 작성한다. 목표를 동사로 쓰면 평가 기준도 자동으로 명확해지기 때문이다.

5. **흐름 검토 및 조정 제안** — 생성된 커리큘럼을 전체 관점에서 읽고 흐름이 튀거나 밀도 편차가 큰 구간을 스스로 지적한다. 수정 제안을 1~3개 덧붙인다.

## 예시

**예시 1:**

입력:
- 과정명: "비전공자를 위한 파이썬 기초"
- 기간: 4주 8회차 (주 2회, 90분)
- 대상: 코딩 경험 없는 직장인
- 목표: 간단한 데이터 처리 스크립트를 혼자 작성할 수 있다

출력 (요약):

| 회차 | 구간 | 주제 | 학습 목표 |
|------|------|------|-----------|
| 1 | 도입 | 파이썬 환경 설정 | 로컬 또는 Colab에서 Hello World를 실행한다 |
| 2 | 도입 | 변수·자료형 | 숫자·문자열·불리언을 변수에 저장하고 출력한다 |
| 3 | 핵심 | 조건문·반복문 | if/for를 사용해 데이터를 필터링한다 |
| 4 | 핵심 | 함수 정의 | 반복 작업을 함수로 묶어 재사용한다 |
| 5 | 핵심 | 파일 입출력 | CSV를 읽고 결과를 파일로 저장한다 |
| 6 | 핵심 | pandas 기초 | DataFrame을 만들고 기본 집계를 수행한다 |
| 7 | 심화 | 실전 데이터 처리 | 제공된 실무 데이터셋을 정리·요약한다 |
| 8 | 통합 | 미니 프로젝트 발표 | 본인 업무 관련 데이터를 스크립트로 처리해 발표한다 |

설계 메모: 5→6회차의 pandas 진입이 갑작스러울 수 있음. 5회차 마지막 10분에 pandas 미리 보기를 추가하는 것을 권장.

---

**예시 2:**

입력:
- 과정명: "팀 리더를 위한 피드백 커뮤니케이션"
- 기간: 하루 워크샵 5세션 (각 80분)
- 대상: 팀장급 리더, 피드백 경험 2년 미만
- 목표: SBI 피드백 모델을 적용해 실제 팀원에게 줄 피드백 메시지를 작성할 수 있다

출력 (요약):

| 세션 | 구간 | 주제 | 활동 |
|------|------|------|------|
| 1 | 도입 | 피드백이 두려운 이유 | 개인 사례 나눔 + 심리 해석 강의 |
| 2 | 핵심 | SBI 모델 이해 | 강의 + 좋은 피드백/나쁜 피드백 분류 실습 |
| 3 | 핵심 | SBI 작성 연습 | 케이스 카드로 피드백 문장 작성 롤플레이 |
| 4 | 심화 | 어려운 상황별 피드백 | 소그룹 케이스 스터디 (지각·성과 미달·갈등) |
| 5 | 통합 | 실전 피드백 작성 | 본인 팀원을 위한 실제 피드백 메시지 완성·공유 |

## 한계 및 주의사항

- **내용 전문성은 검토 필수**: AI는 구조와 흐름을 설계하지만, 각 세션에서 실제로 가르칠 내용의 정확성·깊이·난이도 적절성은 해당 분야 전문가가 검토해야 한다.
- **수강생 반응 예측 불가**: 실제 수업에서 어떤 부분이 막히고 쉽게 넘어가는지는 운영 후 피드백으로만 알 수 있다. 커리큘럼은 1회 운영 후 반드시 업데이트하라.
- **선수 지식 격차 처리 불가**: 수강생 간 실제 수준 차이는 AI가 알 수 없다. 입력받은 "대상" 설명에만 의존하므로, 실제 대상이 다양하면 이를 명확히 명시해야 한다.
- **시간 밀도 현실 검토**: 생성된 각 세션의 분량이 주어진 시간 안에 실제로 소화 가능한지는 강사가 직접 판단해야 한다.
