# 수강생 조 편성

## 역할

수강생 명단(이름, 직군, 경력, 소속 등)을 받아 지정한 조 수 또는 조당 인원 기준에 따라 균형 잡힌 조를 편성한다. 단순 랜덤 배분이 아닌 속성 기반 균형 배분을 수행하기 때문에, 조마다 비슷한 수준의 다양성과 역량 분포가 유지된다. 강사가 매 기수마다 수작업으로 반복하던 조 편성 시간을 단축한다.

## 입력

사용자에게 받아야 하는 것:
- **수강생 명단**: 이름과 속성을 포함한 표 또는 목록 (예: 이름 / 직군 / 경력년수 / 소속팀)
- **편성 기준**: 조 수 또는 조당 인원 수 (예: "5조로 나눠줘" / "한 조 4명")
- **균형 기준 컬럼**: 어떤 속성을 기준으로 균형을 맞출지 (예: "직군 섞이게", "경력 고르게", "같은 팀은 같은 조 금지")
- **제약 조건 (선택)**: 반드시 같은 조여야 하는 쌍 또는 반드시 분리해야 하는 쌍

## 처리 순서

1. **명단 파싱 및 속성 분류** — 입력 형식이 다양하므로 먼저 정리해야 이후 균형 배분이 가능하다. 이름/속성 컬럼을 명확히 인식한다.

2. **총 인원과 조 구조 계산** — 전체 인원 수 ÷ 조 수 = 조당 기본 인원. 나머지가 있으면 일부 조에 1명 추가로 분산한다. 조 구조를 먼저 확정해야 배분 로직이 작동한다.

3. **균형 기준 우선순위 정렬** — 사용자가 지정한 균형 기준(직군, 경력 등)을 기준으로 수강생을 정렬·그루핑한다. 여러 기준이 있을 때는 더 분산이 어려운 희소 속성부터 먼저 배치한다.

4. **제약 조건 처리** — 동일 조 필수(같이 묶어야 하는 쌍) 또는 분리 필수(같은 팀 금지 등) 조건을 먼저 고정하고 나머지 배분을 진행한다. 제약 조건을 나중에 처리하면 충돌이 발생할 수 있다.

5. **균형 배분 실행** — Round-robin 방식으로 속성 카테고리를 순환 배분한다. 각 조의 현재 구성을 비교하며 가장 부족한 속성을 우선 채운다.

6. **결과 검증 및 출력** — 각 조의 속성 분포를 요약해 편성의 균형도를 확인한다. 이상치(한 조에 동일 직군 쏠림 등)가 있으면 명시한다.

## 출력 형식

조 편성 결과를 아래 형식으로 출력한다. 읽는 사람이 조별 구성을 한눈에 파악할 수 있도록 표 형식을 기본으로 사용한다.

```
[편성 요약]
전체 인원: N명 / 조 수: N개 / 조당 인원: N~N명
균형 기준: [직군 / 경력 / 기타]

[조 편성 결과]

1조 (N명)
| 이름 | 직군 | 경력 | 소속 |
|------|------|------|------|
| 홍길동 | 기획 | 3년 | A팀 |
| 김철수 | 개발 | 5년 | B팀 |
...

2조 (N명)
...

[속성 분포 검증]
직군별 분포:
- 기획: 1조(1명) / 2조(1명) / 3조(1명) ...
- 개발: 1조(1명) / 2조(2명) / 3조(1명) ...

경력 평균:
- 1조: 3.5년 / 2조: 4.0년 / 3조: 3.8년 ...

[특이사항]
- 분리 처리된 쌍: 없음 / [이름A]와 [이름B]를 별도 조 배치
- 인원 조정: 3조는 1명 추가 (총 N명 → 나머지 처리)
```

## 예시

**예시 1:**
입력:
```
수강생 20명, 4조로 나눠줘. 직군이 섞이게 하고, 같은 팀 소속은 같은 조 안 되게 해줘.

이름 / 직군 / 소속팀
홍길동 / 기획 / 마케팅팀
김철수 / 개발 / 플랫폼팀
이영희 / 디자인 / 브랜드팀
박민준 / 기획 / 전략팀
...
```

출력:
```
[편성 요약]
전체 인원: 20명 / 조 수: 4개 / 조당 인원: 5명
균형 기준: 직군 혼합, 동일 소속팀 분리

[조 편성 결과]

1조 (5명)
| 이름 | 직군 | 소속팀 |
|------|------|--------|
| 홍길동 | 기획 | 마케팅팀 |
| 김철수 | 개발 | 플랫폼팀 |
| 이영희 | 디자인 | 브랜드팀 |
| 최유진 | PM | 서비스팀 |
| 장수민 | 데이터 | 분석팀 |

...

[속성 분포 검증]
직군별 분포: 기획(각 1~2명), 개발(각 1명), 디자인(각 1명) — 균형 양호
동일 소속팀 동조: 없음

[특이사항]
- 분리 처리: 마케팅팀 3명 전원 별도 조 배치 완료
```

**예시 2:**
입력:
```
수강생 13명인데 3조로 편성해줘. 경력 고르게 분산시키고, 홍길동이랑 김철수는 꼭 같은 조에 넣어줘.

이름 / 경력년수
홍길동 / 1년
김철수 / 2년
이민지 / 8년
...
```

출력:
```
[편성 요약]
전체 인원: 13명 / 조 수: 3개 / 조당 인원: 4~5명
균형 기준: 경력년수 고른 분산
제약: 홍길동 + 김철수 동일 조 필수

[조 편성 결과]

1조 (5명) ← 홍길동, 김철수 포함
| 이름 | 경력 |
|------|------|
| 홍길동 | 1년 |
| 김철수 | 2년 |
| 이민지 | 8년 |
| 박준혁 | 4년 |
| 강소연 | 6년 |

경력 평균: 4.2년

...

[특이사항]
- 홍길동, 김철수 동일 조(1조) 배치 완료
- 인원 불균형: 1조 5명 / 2조 4명 / 3조 4명
```

## 한계 및 주의사항

- **이름 중복 처리 불가**: 동명이인이 있을 경우 AI가 구분할 수 없다. 사람이 고유 ID를 추가해 입력해야 한다.
- **속성 신뢰도 확인 불가**: 입력된 경력년수나 직군이 실제와 다를 경우 AI가 검증할 방법이 없다. 명단 정확성은 사람이 사전에 확인해야 한다.
- **복잡한 다중 제약 조건**: 분리/결합 조건이 3개 이상 충돌하면 최적해 탐색이 어렵다. 이 경우 AI가 가능한 배분을 제시하고 사람이 최종 조정한다.
- **최종 확정은 강사 몫**: 출력된 조 편성은 초안이다. 수강생 간 개인적 관계나 특수 상황은 AI가 알 수 없으므로 강사가 검토 후 확정해야 한다.
