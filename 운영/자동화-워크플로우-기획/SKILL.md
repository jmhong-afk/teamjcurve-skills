# 자동화 워크플로우 기획

## 역할

반복 업무를 구조화된 자동화 기획서로 변환한다. 머릿속에만 있는 "이거 자동화하면 어떨까"를 실제 구현 가능한 트리거-액션-조건 흐름으로 문서화하기 때문에, Make/n8n 같은 도구로 넘어가기 전 설계 시간을 대폭 줄여준다. 모호한 요구사항을 구체적인 실행 단계로 만드는 게 핵심이다.

## 입력

사용자에게 받아야 하는 것:
- 자동화하려는 업무 설명: 현재 어떻게 하고 있는지 자연어로 (예: "매주 월요일 구글폼 응답을 정리해서 노션에 넣고 슬랙으로 알림 보냄")
- 사용 가능한 도구: Make, n8n, Zapier, Google Apps Script 등 (없으면 추천해도 됨)
- 자동화 이유: 줄이고 싶은 시간/실수, 또는 만들고 싶은 결과

## 처리 순서

1. 업무 흐름 파악 — 현재 작업을 시작/중간/끝으로 분해한다. 시작을 모르면 트리거 설정이 불가능하고, 끝을 모르면 성공 기준이 없기 때문에 이 단계가 가장 먼저다.
2. 트리거 정의 — "무엇이 이 자동화를 시작하는가"를 명확히 한다. 스케줄, 이벤트(폼 제출, 이메일 수신), 수동 실행 중 하나로 분류한다.
3. 액션 시퀀스 작성 — 트리거 이후 각 단계를 순서대로 나열한다. 각 액션은 "무엇이 → 어디로" 형태로 기술하면 구현자가 바로 이해할 수 있다.
4. 분기 조건 도출 — "만약 A면 B, 아니면 C" 같은 판단 지점을 찾는다. 여기가 자동화 실패 포인트가 되기 때문에 미리 정의해야 한다.
5. 예외 처리 목록화 — 자동화가 깨질 수 있는 케이스를 열거한다. 실제 구현 시 이 목록이 테스트 케이스가 된다.
6. 구현 우선순위 평가 — 각 단계의 난이도와 효과를 기준으로 "먼저 구현할 것" vs "나중에 추가할 것" 구분한다. 완벽한 자동화보다 빠른 첫 배포가 더 낫다.

## 예시

**예시 1:**
입력: "매주 월요일마다 구글폼 응답을 수동으로 열어서 노션 DB에 복사하고 있어. 팀원 5명한테 슬랙으로 요약 알림도 보내야 함. 30분~1시간 걸림."

출력:
**자동화 이름:** 주간 폼 응답 수집 자동화
**목표:** 수동 복사 및 알림 발송 제거 (주 30~60분 절감)
**현재 소요 시간:** 30~60분/주
**자동화 후 예상 시간:** 0분 (완전 자동)

트리거: 매주 월요일 오전 8시 (스케줄)

액션 시퀀스:
1. Google Sheets(폼 응답) → 신규 행 감지
2. 행 데이터 파싱 → Notion DB 항목 생성
3. 전주 응답 수 집계 → Slack 메시지 포맷 생성
4. Slack 특정 채널 → 메시지 전송

분기 조건:
- IF 신규 응답 0건 → Slack 알림 스킵
- IF 필수 필드 누락 → 운영자에게 별도 DM 알림

예외 처리:
- 시트 컬럼 구조 변경 시 필드 매핑 깨짐 → 필드명 고정 컬럼 사용 권고, 컬럼 위치가 아닌 컬럼명으로 참조

구현 단계:
- 1단계: Sheets → Notion 복사 흐름만
- 2단계: Slack 알림 + 분기 조건 추가

추천 도구: Make.com (Google Sheets, Notion, Slack 모두 네이티브 연동 지원)

---

**예시 2:**
입력: "교육 신청자가 들어오면 환영 이메일 보내고 노션에 수강생 DB 추가하고 싶어. 지금은 신청 확인 후 수동으로 다 하고 있음."

출력:
**자동화 이름:** 교육 신청 접수 자동화
**목표:** 신청 접수부터 온보딩까지 수동 작업 제거
**현재 소요 시간:** 건당 5~10분
**자동화 후 예상 시간:** 0분

트리거: 구글폼 신청서 제출 (이벤트)

액션 시퀀스:
1. 폼 응답 파싱 → 수강생 정보 추출 (이름, 이메일, 기수)
2. Notion 수강생 DB → 신규 항목 생성
3. Gmail → 환영 이메일 발송 (템플릿 기반)

분기 조건:
- IF 동일 이메일이 Notion에 이미 존재 → 항목 생성 스킵 + 운영자에게 중복 알림
- IF 기수 미선택 → 이메일 발송 보류 + 수동 확인 요청

예외 처리:
- 필수 필드 미입력 → 자동화 스킵 후 운영자 이메일로 원본 응답 전달
- 이메일 발송 실패 → Make 오류 로그 기록 + 재시도 1회

추천 도구: Make.com (Google Forms 트리거가 Zapier보다 안정적)

## 한계 및 주의사항

- AI는 실제 API 연동 가능 여부를 모른다. 연동 가능 여부는 Make/n8n 앱 목록에서 직접 확인해야 한다.
- 예외 케이스는 사전 열거할 수 있지만, 실제 운영 중 발생하는 엣지 케이스는 구현 후 테스트로만 발견된다.
- 추천 도구는 일반적 기준이다. 팀 내 기존 도구 스택, 비용, 계정 상황에 따라 최적 선택이 달라진다.
- 설계 후 사람이 직접 소규모 테스트(더미 데이터로 1회 실행)를 해야 안정성을 확인할 수 있다.
