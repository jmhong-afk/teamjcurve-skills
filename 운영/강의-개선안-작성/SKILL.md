# 강의 개선안 작성

## 역할

수강생 피드백을 그대로 나열하거나 요약하는 데 그치지 않고, "다음에 무엇을 어떻게 바꿀 것인가"라는 실행 관점의 개선안을 만든다. 강의 기획자가 다음 기수 준비 시 즉시 참고할 수 있도록, 우선순위와 담당 레이어(강사/운영/커리큘럼)가 명확한 액션 리스트로 출력한다.

## 입력

사용자에게 받아야 하는 것:
- 피드백 원문 또는 요약: 설문 응답, 수강생 자유 의견, 강사 메모 등 형식 무관 (예: "실습 시간이 너무 짧았다", "3주차 내용이 너무 어려웠다" 등)
- 강의 기본 정보: 강의명, 대상 수준(입문/중급 등), 총 주차 수 (예: "AI 활용 실무 과정, 입문, 4주")
- 개선 범위 (선택): 커리큘럼 내용 / 운영 방식 / 강사 전달력 / 과제·실습 중 집중할 영역 (없으면 전체 검토)

## 처리 순서

1. 피드백 클러스터링 — 비슷한 내용의 피드백을 묶어 반복 패턴을 파악한다. 개별 의견이 아닌 경향을 봐야 개선 우선순위가 명확해지기 때문이다.

2. 레이어 분류 — 각 피드백 클러스터를 "커리큘럼", "운영", "강사 전달"로 분류한다. 레이어가 달라야 개선 담당자가 달라지므로, 섞이면 실행이 흐려진다.

3. 개선 가능 여부 판단 — 각 항목이 다음 기수에 반영 가능한지(즉시/검토 필요/불가) 판단한다. 불가 항목은 이유를 명시해 혼선을 줄인다.

4. 액션 플랜 작성 — 반영 가능한 항목에 대해 구체적 변경 내용을 기술한다. "~을 개선한다"가 아니라 "3주차 실습 시간을 40분→70분으로 늘리고, 강사 설명을 20분 압축한다"처럼 실행 단위로 작성한다.

5. 우선순위 정렬 — 수강생 언급 빈도 + 강의 목표 달성에 미치는 영향을 기준으로 High/Mid/Low로 분류한다. 모든 것을 동시에 바꾸려다 아무것도 못 바꾸는 것을 방지하기 위해서다.

## 출력 형식

```
# [강의명] [기수/날짜] 개선안

## 피드백 요약 (클러스터)
| 주제 | 빈도 | 대표 의견 |
|------|------|----------|
| 실습 시간 부족 | 12건 | "실습을 더 하고 싶었어요" |
| 3주차 난이도 | 8건 | "갑자기 어려워졌어요" |
| ...  | ...  | ... |

## 개선 액션 플랜

### High Priority
- [ ] [레이어] 변경 내용 — 근거 (피드백 클러스터)
- [ ] [레이어] 변경 내용 — 근거

### Mid Priority
- [ ] [레이어] 변경 내용 — 근거

### Low Priority (검토 후 결정)
- [ ] [레이어] 변경 내용 — 근거

## 반영 불가 항목
- 항목: 이유

## 다음 기수 체크리스트
강의 전 확인해야 하는 변경 사항 요약 (3~5개 bullet)
```

레이어 표기: [커리큘럼], [운영], [강사]

## 예시

**예시 1:**
입력:
- 강의명: AI 활용 실무 입문 과정, 4주, 입문
- 피드백: "실습 시간이 짧아서 아쉬웠다(15명)", "3주차가 갑자기 어려웠다(10명)", "강사님 설명이 친절했다(20명)", "과제 피드백이 늦었다(5명)"

출력:
```
# AI 활용 실무 입문 과정 개선안

## 피드백 요약 (클러스터)
| 주제 | 빈도 | 대표 의견 |
|------|------|----------|
| 실습 시간 부족 | 15건 | "실습 시간이 짧아서 아쉬웠다" |
| 3주차 난이도 급상승 | 10건 | "갑자기 어려워졌어요" |
| 과제 피드백 지연 | 5건 | "피드백이 늦어서 다음 진도 때 반영 못했다" |

## 개선 액션 플랜

### High Priority
- [ ] [커리큘럼] 3주차 내용을 2개 회차로 분리하거나 2주차 후반에 징검다리 내용 추가 — 난이도 급상승 피드백 10건
- [ ] [커리큘럼] 회차당 실습 시간 20분 추가, 강사 설명 압축 — 실습 부족 피드백 15건

### Mid Priority
- [ ] [운영] 과제 피드백 마감 기준 명문화 (제출 후 48시간 이내) — 지연 피드백 5건

### Low Priority
- [ ] [운영] 실습 보조 자료(치트시트) 제공 검토

## 다음 기수 체크리스트
- 3주차 커리큘럼 분리 여부 최종 확인
- 회차별 타임라인 재배분 (실습 +20분)
- 과제 피드백 SLA 운영팀 공유
```

**예시 2:**
입력:
- 강의명: 퍼스널 브랜딩 워크샵, 1일, 중급
- 피드백: "오전은 좋았는데 오후가 너무 빨리 진행됐다(8명)", "실제 사례가 더 있었으면(6명)", "그룹 활동이 너무 짧았다(7명)"

출력:
```
# 퍼스널 브랜딩 워크샵 개선안

## 피드백 요약 (클러스터)
| 주제 | 빈도 | 대표 의견 |
|------|------|----------|
| 오후 진행 속도 | 8건 | "오후가 너무 빨리 진행됐다" |
| 그룹 활동 시간 부족 | 7건 | "그룹 활동이 짧았다" |
| 사례 보강 요청 | 6건 | "실제 사례가 더 있었으면" |

## 개선 액션 플랜

### High Priority
- [ ] [운영] 오후 타임라인 재설계 — 강의 10분 축소, 그룹 활동 15분 추가
- [ ] [커리큘럼] 실제 사례 슬라이드 3~4개 추가 (업종별 1개씩)

### Mid Priority
- [ ] [강사] 오후 시작 전 간단한 에너지 활성 활동 추가 (5분)

## 다음 기수 체크리스트
- 오후 타임라인 수정본 강사와 사전 공유
- 사례 슬라이드 제작 완료 여부 확인
- 그룹 활동 발표 시간 별도 배정
```

## 한계 및 주의사항

- 피드백 수가 적으면(5건 미만) 경향 파악이 어렵다 — 소수 의견임을 명시하고, 사람이 추가 판단할 것
- AI는 강의 현장 맥락(강사의 실제 퍼포먼스, 수강생 특성, 예산 제약)을 모른다 — 출력된 개선안은 반드시 강사·운영팀과 논의 후 확정할 것
- 커리큘럼 구조 변경은 기존 일정·자료와 충돌할 수 있다 — 사람이 실행 가능성 최종 검토 필요
