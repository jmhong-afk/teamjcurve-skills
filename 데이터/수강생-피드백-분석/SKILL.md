# 수강생 피드백 분석

## 역할

수강생이 남긴 다양한 형태의 피드백(설문, 후기, 쪽글, 댓글)을 체계적으로 분류하고 패턴을 추출해 다음 기수 개선에 바로 쓸 수 있는 인사이트를 생성한다. 사람이 수백 개 응답을 읽으며 직감으로 요약하면 놓치기 쉬운 반복 패턴과 소수 의견의 신호를 모두 포착하는 것이 목적이다.

## 입력

사용자에게 받아야 하는 것:
- **피드백 원문**: 설문 응답, 후기, 댓글 등 텍스트 묶음 (복사-붙여넣기 또는 파일 경로). 예: "3기 수료 후기 구글폼 응답 87건"
- **프로그램 맥락** (선택): 프로그램 이름, 기간, 주요 커리큘럼 목표. 맥락이 있으면 더 정확한 카테고리 분류 가능. 예: "12주 스타트업 코호트 프로그램, 목표: 창업 실행력"
- **분석 목적** (선택): "다음 기수 커리큘럼 개선", "강사 피드백", "마케팅 증언 발굴" 등. 목적이 없으면 전체 종합 분석으로 진행.

## 처리 순서

1. **피드백 전처리** — 중복, 의미 없는 응답(예: "없음", "모르겠음"), 오탈자를 걸러낸다. 전처리 없이 분석하면 노이즈가 카테고리를 오염시킨다.

2. **카테고리 분류** — 각 피드백을 아래 기본 카테고리로 분류한다. 입력된 프로그램 맥락에 따라 카테고리를 조정할 것.
   - 커리큘럼/내용
   - 강사/퍼실리테이터
   - 운영/행정 (일정, 공지, 환경)
   - 동료/커뮤니티
   - 개인 성장/성과
   - 기타

3. **감정 레이블링** — 각 카테고리 내 응답을 긍정/부정/제안/중립으로 표시한다. 부정과 제안을 합산하면 개선 우선순위 가중치가 된다.

4. **패턴 추출** — 동일한 의미로 반복되는 응답을 묶고, 묶음별 빈도수를 센다. 빈도수가 높은 순으로 정렬하는 이유는 운영팀이 가장 먼저 볼 항목이 무엇인지 명확히 하기 위함이다.

5. **소수 신호 포착** — 1~2건만 언급됐지만 임팩트가 클 수 있는 의견을 별도로 플래그한다. 다수결로만 운영하면 핵심 변화를 놓친다.

6. **액션 아이템 도출** — 분석 결과를 바탕으로 "다음 기수에 바로 적용 가능한" 구체적 개선안을 작성한다. 추상적인 제안("커리큘럼을 강화한다") 대신 실행 가능한 수준("3주차 실습 시간을 60분에서 90분으로 늘린다")으로 작성한다.

## 출력 형식

```
# [프로그램명] [기수] 수강생 피드백 분석 결과

## 요약
- 분석 대상: N건 (유효 응답 N건)
- 분석 기간: YYYY-MM-DD
- 전체 감정 분포: 긍정 N% / 부정 N% / 제안 N% / 중립 N%

## 카테고리별 주요 패턴

### 커리큘럼/내용 (N건)
- [긍정] [패턴 요약] — N건
- [부정] [패턴 요약] — N건
- [제안] [패턴 요약] — N건

### 강사/퍼실리테이터 (N건)
...

### 운영/행정 (N건)
...

(카테고리 반복)

## 소수 의견 신호
- "[원문 요약]" — 1건, 플래그 이유: [왜 중요한지]

## 대표 인용
긍정: "[가장 강한 긍정 응답 원문 발췌]"
개선 필요: "[가장 구체적인 불만/제안 원문 발췌]"

## 액션 아이템 (우선순위순)
1. [카테고리] [구체적 개선안] — 근거: [어떤 피드백에서 나왔는지]
2. ...
3. ...

## 다음 기수 전달 메모
[운영팀/강사에게 전달할 핵심 한 줄 요약]
```

## 예시

**예시 1:**
입력:
```
프로그램: J커브 스타트업 코호트 4기 (8주)
피드백 17건 발췌:
- "실습 시간이 너무 짧아서 아쉬웠어요"
- "강사님이 질문에 바로바로 답변해 주셔서 좋았습니다"
- "조별 활동이 제일 도움됐어요"
- "실습 시간이 부족했습니다"
- "피드백 세션 더 늘려주세요"
- "커리큘럼 자체는 완성도 높았어요"
- "온라인 자료 링크가 자꾸 깨져 있었어요"
```

출력 (요약):
```
## 카테고리별 주요 패턴

### 커리큘럼/내용 (5건)
- [긍정] 커리큘럼 완성도 높다는 평가 — 1건
- [제안] 실습 시간 확대 요청 — 2건
- [제안] 피드백 세션 확대 요청 — 1건

### 강사/퍼실리테이터 (1건)
- [긍정] 즉각적인 질의응답 만족 — 1건

### 운영/행정 (1건)
- [부정] 온라인 자료 링크 오류 — 1건

### 동료/커뮤니티 (1건)
- [긍정] 조별 활동 효과적 — 1건

## 액션 아이템
1. [커리큘럼] 실습 세션을 회차당 15분 추가 확보 — 근거: 실습 시간 부족 2건
2. [운영] 매 기수 시작 전 공유 링크 일괄 검수 프로세스 도입 — 근거: 링크 오류 1건
```

**예시 2:**
입력: 엑셀 복사 붙여넣기 형태의 구글폼 응답 50건 (단답형 + 5점 척도 혼합)
출력: 카테고리 분류 + 척도 평균값 포함 분석 리포트, 마케팅에 쓸 수 있는 증언 문구 3개 별도 추출

## 한계 및 주의사항

- **감정 판단의 한계**: 한국어 맥락 특유의 완곡한 표현("그냥 그랬어요", "나쁘지 않았어요")은 긍정/부정 오분류 가능. 사람이 최종 검토 필수.
- **개인 식별 정보**: 피드백에 이름, 연락처가 포함된 경우 Claude에 그대로 넣지 말 것. 사전에 익명화 처리.
- **액션 아이템 실현 가능성 판단 불가**: Claude는 내부 예산·일정 제약을 모른다. 아이템별 실현 가능성 판단은 운영팀이 직접 해야 한다.
- **척도형(1-5점) 데이터 처리**: 텍스트만 붙여넣으면 수치 집계 불가. 수치가 있으면 함께 제공해야 평균/분포 포함 가능.
