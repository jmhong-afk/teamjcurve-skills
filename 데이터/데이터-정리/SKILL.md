# 데이터 정리

## 역할

반복적으로 발생하는 날것의 데이터(설문 응답, 출석 명단, 피드백 목록, 참가자 정보 등)를 받아 정제·분류·요약하는 작업을 자동화한다. 사람이 수동으로 엑셀을 편집하거나 응답을 하나씩 분류하는 데 소요되는 시간을 줄이고, 데이터에서 바로 인사이트를 뽑아낼 수 있도록 돕는다.

## 입력

사용자에게 받아야 하는 것:
- **원본 데이터**: 텍스트로 붙여넣은 CSV, 탭 구분 표, 줄바꿈으로 나열된 응답 목록 등 (예: 구글 폼 내보내기 텍스트, 노션 표 복사본)
- **목적/맥락**: 이 데이터를 왜 정리하는지 한 문장 (예: "5기 수강생 사전 설문 결과 파악용", "3월 출석률 계산용")
- **분류 기준** (선택): 특정 칼럼을 기준으로 묶거나 필터링해야 한다면 명시 (예: "직군별로 분류", "미응답 제외")

## 처리 순서

1. **데이터 파악** — 칼럼 구조, 행 수, 데이터 타입(텍스트/숫자/날짜)을 먼저 파악해야 이후 단계에서 적합한 정제 방식을 선택할 수 있다.
2. **정제** — 중복 행 제거, 공백·오탈자 통일, 누락값 표시(N/A 처리). 원본을 바꾸는 것이 아니라 정제된 버전을 별도로 제시하기 때문에 사람이 원본과 비교할 수 있다.
3. **분류/집계** — 목적과 분류 기준에 따라 카테고리를 나누고 빈도·비율을 계산한다. 수치로 표현할 수 있는 것은 수치화하는 것이 판단 근거를 명확히 하기 때문이다.
4. **인사이트 도출** — 수치에서 패턴, 편중, 이상값을 찾아 불릿으로 정리한다. 숫자만 나열하면 의사결정에 시간이 걸리므로 "이 데이터가 의미하는 것"을 한 줄씩 해석해준다.
5. **다음 액션 제안** — 인사이트를 바탕으로 사람이 취할 수 있는 다음 단계를 1~3가지 제시한다. 데이터 정리의 목적은 결국 행동이기 때문이다.

## 출력 형식

```
## 데이터 정리 결과

### 기본 정보
- 총 데이터 수: N건
- 유효 데이터: N건 (중복/누락 제외 후)
- 정제 내역: [중복 N건 제거, 공백 통일 등]

### 정제 데이터 (테이블)
| 칼럼1 | 칼럼2 | 칼럼3 |
|-------|-------|-------|
| ...   | ...   | ...   |

### 집계 요약
| 항목 | 건수 | 비율 |
|------|------|------|
| ...  | ...  | ...  |

### 주요 인사이트
- [패턴/발견 1]: 구체적 수치와 함께 해석
- [패턴/발견 2]: 구체적 수치와 함께 해석
- [이상값/주목 포인트]: ...

### 다음 액션 제안
1. ...
2. ...
```

## 예시

**예시 1:**
입력:
```
목적: 5기 수강생 직군 파악
데이터:
이름,직군,경력
김민준,마케터,3년
이서연,디자이너,1년
박지호,마케터,5년
최유나,개발자,2년
정다은,마케터,3년
이서연,디자이너,1년  ← 중복
```
출력:
```
## 데이터 정리 결과

### 기본 정보
- 총 데이터 수: 6건
- 유효 데이터: 5건 (중복 1건 제거)
- 정제 내역: 이서연(디자이너, 1년) 중복 행 1건 제거

### 집계 요약
| 직군 | 건수 | 비율 |
|------|------|------|
| 마케터 | 3 | 60% |
| 디자이너 | 1 | 20% |
| 개발자 | 1 | 20% |

### 주요 인사이트
- 마케터 비중이 60%로 가장 높아 마케팅 실습 비중을 상대적으로 높게 설계하는 것이 적합함
- 평균 경력 약 2.8년으로 기초보다는 중급 난이도가 적합함

### 다음 액션 제안
1. 마케터 중심 케이스스터디 비중 확대 검토
2. 직군 혼합 조 편성 시 마케터+개발자 조합 고려
```

**예시 2:**
입력:
```
목적: 3월 워크샵 출석 현황 파악
데이터:
3/5 출석: 김민준, 이서연, 박지호, 최유나
3/12 출석: 김민준, 박지호, 정다은
3/19 출석: 이서연, 최유나, 정다은, 박지호
```
출력:
```
## 데이터 정리 결과

### 기본 정보
- 총 인원: 5명
- 회차: 3회 (3/5, 3/12, 3/19)

### 출석 집계
| 이름 | 3/5 | 3/12 | 3/19 | 출석률 |
|------|-----|------|------|--------|
| 김민준 | O | O | X | 67% |
| 이서연 | O | X | O | 67% |
| 박지호 | O | O | O | 100% |
| 최유나 | O | X | O | 67% |
| 정다은 | X | O | O | 67% |

### 주요 인사이트
- 전체 평균 출석률 73% — 목표치(80%) 대비 7%p 부족
- 박지호만 3회 전 출석, 나머지는 모두 2회

### 다음 액션 제안
1. 결석자 대상 보강 자료 배포 여부 결정 필요
2. 4월 일정 공지 시 사전 참석 확인 받는 것 권장
```

## 한계 및 주의사항

- **수식·계산 검증 불가**: AI가 숫자를 집계하지만 복잡한 조건부 계산은 오류가 생길 수 있다. 중요한 수치는 엑셀/스프레드시트로 재검증할 것.
- **원본 데이터 보존 책임은 사람에게**: AI는 정제된 버전만 제시하며 원본을 변경하지 않는다. 원본 파일은 반드시 따로 보관해둘 것.
- **개인정보 포함 데이터**: 이름, 연락처 등이 포함된 경우 외부 AI 서비스에 전달하기 전 정책 확인 필요.
- **인사이트 해석의 맥락 한계**: AI는 데이터 내 수치 패턴을 파악하지만 조직 내부 맥락(팀 사정, 히스토리)은 알 수 없다. 인사이트의 타당성은 사람이 최종 판단할 것.
