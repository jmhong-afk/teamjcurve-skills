# 프로그램 성과 보고서

## 역할

교육 프로그램 한 기수가 끝날 때마다 반복적으로 작성해야 하는 성과 보고서를 구조화해 준다. 원시 운영 데이터(참여율, 수료율, 피드백 점수, 주요 성과물)를 받아 이해관계자가 빠르게 읽을 수 있는 보고서 형식으로 전환한다. 숫자 나열이 아닌 "왜 이 결과가 나왔는지"와 "다음에 무엇을 바꿔야 하는지"까지 포함하기 때문에 단순 집계보다 훨씬 높은 가치를 전달한다.

## 입력

사용자에게 받아야 하는 것:
- 프로그램 기본 정보: 프로그램명, 기수, 운영 기간, 목표 (예: "AI 리터러시 부트캠프 3기, 2026년 2월~3월, 직장인 대상 8주 과정")
- 참여 수치: 지원자 수, 선발 수, 수료 수, 중도 이탈 수 (예: "지원 142명, 선발 40명, 수료 34명, 이탈 6명")
- 학습 성과 데이터: 만족도 평균, NPS 점수, 최종 프로젝트 완성률, 주요 성과물 요약 (예: "만족도 4.6/5, NPS +62, 프로젝트 완성률 85%")
- 운영 이슈 메모: 이번 기수에서 특이했던 점, 잘된 것, 어려웠던 것 (예: "3주차 참여율 급감, 멘토링 세션 반응 좋음")
- 보고서 수신자: 누가 읽는지 (예: "내부 운영팀", "외부 스폰서", "팀 회고용")

## 처리 순서

1. 기본 지표 계산 및 정리 — 입력된 숫자를 기반으로 수료율, 이탈률, 만족도 등급 등 파생 지표를 먼저 계산한다. 해석보다 수치가 먼저 정확해야 그 위에 서술이 성립하기 때문이다.

2. 스토리라인 구성 — "이번 기수는 어땠는가"를 한 줄로 요약하는 핵심 메시지를 먼저 잡는다. 수신자가 보고서 첫 단락만 읽어도 핵심을 파악할 수 있어야 하기 때문이다.

3. 섹션별 서술 작성 — 운영 현황 → 학습 성과 → 참여 패턴 → 이슈 및 특이사항 순으로 작성한다. 이 순서는 "무슨 일이 있었나" → "결과는 어떤가" → "왜 그랬나" 흐름으로 독자가 자연스럽게 따라올 수 있기 때문이다.

4. 인사이트 및 개선 제언 작성 — 수치와 이슈를 바탕으로 다음 기수에서 바꿔야 할 것 2~3가지를 근거와 함께 제시한다. 보고서의 실질적 가치는 과거 기록이 아니라 미래 의사결정 지원에 있기 때문이다.

5. 수신자 맞춤 조정 — 내부 운영팀용은 운영 이슈와 개선 액션 중심, 외부 스폰서용은 임팩트 수치와 수강생 성과 중심, 팀 회고용은 학습과 실험 결과 중심으로 톤과 강조점을 조정한다.

## 출력 형식

보고서는 스캔하기 쉬운 구조가 핵심이다. 바쁜 이해관계자가 3분 안에 핵심을 파악할 수 있도록 섹션 헤딩과 굵은 수치를 전면에 배치한다.

```

운영 기간: YYYY.MM.DD ~ YYYY.MM.DD
```

## 예시

**예시 1:**
입력: "AI 리터러시 부트캠프 3기, 2026년 2~3월, 직장인 40명 선발. 수료 34명(이탈 6명). 만족도 4.6/5, NPS +62, 프로젝트 완성률 85%. 3주차 참여율 급감 있었고 멘토링 세션 반응 매우 좋았음. 내부 운영팀용."
출력:
```
# AI 리터러시 부트캠프 3기 성과 보고서
운영 기간: 2026.02.03 ~ 2026.03.28

## 한 줄 요약
수료율 85%, NPS +62로 전기 대비 고른 성과를 달성했으나, 3주차 참여율 저하가 반복 과제로 확인됨.

## 운영 현황
| 지표 | 수치 |
|------|------|
| 선발 | 40명 |
| 수료 | 34명 (수료율 85%) |
| 이탈 | 6명 (이탈률 15%) |

## 학습 성과
- 만족도: 4.6 / 5.0
- NPS: +62
- 최종 프로젝트 완성률: 85%

## 참여 패턴 및 이슈
- 잘된 것: 멘토링 세션 참여 만족도 높음, 자발적 스터디 그룹 3개 자생
- 어려웠던 것: 3주차 주간 참여율 62%로 급감 (전주 대비 -23%p)
- 특이사항: 직장인 과제 제출 마감 탄력 운영 요청 다수

## 다음 기수 개선 제언
1. 3주차 콘텐츠 부하 분산 — 실습 과제량이 3주차에 집중되어 참여율 저하 유발. 2~4주 균등 배분 검토
2. 멘토링 세션 횟수 확대 — 가장 높은 만족도 요인으로 확인. 현 2회에서 3회로 증설 권장
3. 과제 마감 유연화 정책 공식화 — 직장인 특성상 주말 집중 제출 패턴. 마감 기준 재설정 필요
```

**예시 2:**
입력: "스타트업 마케팅 집중 과정 1기, 온라인 4주. 지원 89명, 선발 25명, 수료 21명. 만족도 4.3/5, 프로젝트 제출률 92%. 커뮤니티 활성화가 예상보다 저조했음. 외부 스폰서 보고용."
출력:
```
# 스타트업 마케팅 집중 과정 1기 성과 보고서

## 한 줄 요약
수료율 84%, 프로젝트 완성률 92%로 1기 목표치를 달성했으며 참가자 만족도가 업계 평균을 상회함.

## 운영 현황
| 지표 | 수치 |
|------|------|
| 지원 | 89명 (선발 경쟁률 3.6:1) |
| 선발 | 25명 |
| 수료 | 21명 (수료율 84%) |

## 학습 성과
- 만족도: 4.3 / 5.0
- 최종 프로젝트 제출률: 92%
- 주요 성과물: 수료생 21명 전원 실제 마케팅 캠페인 기획서 완성 및 피드백 완료

## 다음 기수 개선 제언
1. 커뮤니티 온보딩 구조화 — 1기에서 자발적 네트워킹 활성화 저조. 오리엔테이션 내 팀 빌딩 세션 추가 권장
2. 선발 기준 공개 — 경쟁률 3.6:1 수준에서 지원자 기대치 관리를 위한 선발 기준 사전 안내 필요
```

## 한계 및 주의사항

- 수치 해석의 맥락은 AI가 추론하는 것이므로, 산업 벤치마크나 전기 실제 수치와 비교가 필요한 경우 사람이 직접 데이터를 추가 제공해야 한다.
- "잘된 것/어려웠던 것" 서술은 입력된 운영 메모를 기반으로 작성하므로, 입력이 빈약하면 출력도 얕아진다. 운영 중 실시간으로 메모를 축적하는 습관이 보고서 품질을 결정한다.
- 외부 스폰서용 보고서는 민감한 이탈 이유, 내부 운영 문제 등을 포함하지 않도록 사람이 최종 검토해야 한다.
- 수치 계산(수료율, NPS 등)은 입력값을 기반으로 계산하므로 입력 숫자의 정확성은 사람이 확인해야 한다.
